# Contributing to Monitoring Scripts Repository

Thank you for your interest in contributing to this project! We welcome contributions from the community to improve and expand the functionality of our monitoring scripts. This guide will help you get started with your contributions.

## How to Contribute

### 1. Fork the Repository

Start by forking the repository to your GitHub account:

1. Navigate to the repository's main page.
2. Click the **Fork** button in the upper right corner.
3. Clone the forked repository to your local machine:

   ```bash
   git clone https://github.com/your-username/repository-name.git
   cd repository-name
   ```

### 2. Create a New Branch

Always create a new branch for your work to keep your changes separate from the main codebase:

```bash
git checkout -b my-new-feature
```

### 3. Make Your Changes

Make the necessary changes or additions to the repository. Be sure to follow any relevant coding standards or guidelines outlined in this file.

- Follow the structure and organization of the repository.
- Write clear, concise, and well-documented code.
- Test your changes thoroughly before submitting them.

### 4. Commit Your Changes

When you're ready to commit your changes, use descriptive and meaningful commit messages:

```bash
git add .
git commit -m "Add a feature to monitor CPU usage"
```

### 5. Push Your Changes to GitHub

Push your changes to your forked repository:

```bash
git push origin my-new-feature
```

### 6. Submit a Pull Request

Once your changes are pushed, open a pull request:

1. Go to the original repository on GitHub.
2. Click the **Pull Requests** tab, then click **New Pull Request**.
3. Select the branch you want to merge from and the target branch.
4. Describe your changes in the pull request.

### 7. Address Feedback

If the maintainers provide feedback on your pull request, make sure to address it. You can push additional commits to your branch, and the pull request will be updated automatically.

## Code Style Guidelines

Please follow these guidelines when contributing:

- Use **Bash** standards for scripting.
- Comment your code appropriately to explain complex logic.
- Keep the scripts modular and reusable.
- Use descriptive variable names.

## Reporting Bugs or Issues

If you find a bug, error, or have a feature request, please open an issue on GitHub. Be sure to include:

- A descriptive title.
- Steps to reproduce the issue (if applicable).
- Environment details (e.g., AWS, operating system).
- Suggested improvements or feature details (if applicable).

## Adding New Scripts

We welcome new monitoring scripts! If you'd like to add a new script to the repository, follow these steps:

1. Create a new directory for your script within the repository.
2. Add your script along with a `README.md` explaining:
   - How to use the script.
   - Example usages.
   - Any necessary environment setup.
3. Follow the commit and pull request process outlined above.
